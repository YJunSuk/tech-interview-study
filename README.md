# 🚀 [CS 전공지식] 8주 완성 면접 스터디

> **"기초가 탄탄한 개발자가 되기 위한 8주간의 여정"**

---

### 📋 스터디 규칙
* **[RULE 01]** 스터디 불참 시 **미리 공지**하기
* **[RULE 02]** `Notion`에 학습 기록을 남기고, 종료 후 `GitHub`에 노션 링크와 함께 커밋하기
* **[RULE 03]** 스터디 전까지 **GitHub Issue**에 질문 최소 2개 등록

### 📅 일정 및 방식
* **기간**: 총 8주 (2개월)
* **시간**: 주 2회 (요일/시간 협의)
* **방식**: **오프라인**
* **진행**: 사전 학습 ➡️ 질문 공유 ➡️ **랜덤 화이트보드 브리핑**

---

- **1회차 (1.1.1 ~ 1.1.5):**
    - `핵심`: **"왜 쓰는가?"** 에 집중. 싱글톤 멀티스레드 이슈, 전략/프록시 패턴의 실제 사례(Spring, React 등)
- **2회차 (1.1.6 ~ 1.2.4):**
    - `핵심`: MVC/MVP/MVVM 차이 명확화. 객체지향 vs 함수형 프로그래밍 비교

#### **2주차: 네트워크 기초와 물리적 계층**
- **3회차 (2.1 ~ 2.2):**
    - `핵심`: OSI 7계층 vs TCP/IP 4계층. **PDU 캡슐화** 흐름 파악
- **4회차 (2.3 ~ 2.4):**
    - `핵심`: 라우터/스위치 차이, **ARP** 동작 과정 시나리오 복기

#### **3주차: HTTP와 보안**
- **5회차 (2.5.1 ~ 2.5.5):**
    - `핵심`: **[면접 단골]** HTTP/1.1 HOLB ~ HTTP/3 QUIC (성능 개선 관점)
- **6회차 (네트워크 총정리):**
    - `핵심`: **"주소창에 google.com을 치면 일어나는 일"** 모의 면접

#### **4주차: 운영체제와 메모리**
- **7회차 (3.1 ~ 3.2):**
    - `핵심`: 가상 메모리, 페이지 교체 알고리즘, 메모리 계층 구조의 필요성
- **8회차 (중간 점검):**
    - `핵심`: 한 달간 배운 내용 중 가장 어려웠던 개념 리마인드

- **9회차 (3.3.1 ~ 3.3.4):**
