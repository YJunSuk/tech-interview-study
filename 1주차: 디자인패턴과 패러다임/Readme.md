### **1주차: 디자인 패턴과 패러다임**

#### **1회차 (1.1.1 ~ 1.1.5):**
- 1장. 디자인 패턴과 프로그래밍 패러다임

- 1.1 디자인 패턴
    - 1.1.1 싱글톤 패턴
    - 1.1.2 팩토리 패턴
    - 1.1.3 전략 패턴
    - 1.1.4 옵저버 패턴
    - 1.1.5 프록시 패턴과 프록시 서버
- **핵심:** 각 패턴을 "왜 쓰는가?"에 집중하세요. 특히 싱글톤의 멀티스레드 이슈나, 전략 패턴과 프록시 패턴의 실제 라이브러리 적용 사례(예: Spring, Vue/React 등)를 찾아보면 좋습니다.
#### **2회차 (1.1.6 ~ 1.2.4):**
- 1.1 디자인 패턴
    - 1.1.6 이터레이터 패턴
    - 1.1.7 노출모듈 패턴
    - 1.1.8 MVC 패턴
    - 1.1.9 MVP 패턴
    - 1.1.10 MVVM 패턴

- 1.2 프로그래밍 패러다임
    - 1.2.1 선언형과 함수형 프로그래밍
    - 1.2.2 객체지향 프로그래밍
    - 1.2.3 절차형 프로그래밍
    - 1.2.4 패러다임의 혼합
- **핵심:** MVC/MVP/MVVM의 차이점을 명확히 구분하세요. 또한 객체지향과 함수형 프로그래밍의 특징을 비교하며, 현대 프레임워크가 이를 어떻게 혼합하는지 논의해 보세요.
